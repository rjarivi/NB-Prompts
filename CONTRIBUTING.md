The following is a set of guidelines for contributing to Nano Banana Prompts. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs

- **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/rjarivi/nano-banana-prompts/issues).
- If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/rjarivi/nano-banana-prompts/issues/new). Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements

- Open a new issue and describe your proposal clearly.
- Explain why this enhancement would be useful to most users.

### Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code lints.
5. Update the documentation (README.md, etc.) if appropriate.

## Styleguides

### JavaScript

- Use modern ES6+ syntax.
- Keep functions small and focused.
- Comment complex logic.

### HTML/CSS

- We use Tailwind CSS via CDN for styling.
- Ensure the layout remains responsive.
- Maintain the dark mode/glassmorphism aesthetic.
